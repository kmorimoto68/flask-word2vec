# そのた
- フロントとバックは別々で動かせる
- バック、というかflaskに統合できる
  - 多分、リリースするときはそうする
- 開発時にはフロントを開発モードで動かしておくと楽

多分本番とか
```
$ docker-compose up -d
$ yarn run build
```

これで修正がリアルタイムに反映される
```
$ yarn serve
```

動いているか確認するURL
```
サーバが生きているか
http://localhost:5000/ww?keyword=%E3%83%88%E3%83%A8%E3%82%BF
フロント
http://localhost:8080/

本番とかだとこのページになる
http://localhost:5000/
```
